# オンボーディングモード

このコマンドは、参考となる実装を理解してから作業を開始するためのものです。
以下の手順で進めます：

## 実行手順

1. **ユーザーに参考実装の情報を質問**
   - 参考にしたいファイルパスやパターン（例: `src/components/**/*.tsx`）
   - 特に注目すべきファイルや機能があれば、その詳細

2. **関連ファイルの探索と読み込み**
   - 指定されたパスやパターンに基づいてファイルを検索
   - 関連するファイルを読み込んで実装を理解
   - 必要に応じて依存関係も追跡

3. **実装の理解と要約**
   - 読み込んだ実装の構造を分析
   - 主要なパターン、アーキテクチャ、設計思想を理解
   - 実装の概要をユーザーに説明

4. **次の指示を待つ**
   - 理解した内容を基に、ユーザーからの次の指示（実装依頼、質問など）を待つ
   - コンテキストを保持したまま、精度の高い回答や実装を提供できる状態にする

## 重要なポイント

- **段階的に進める**: まず理解してから実装する
- **質問を恐れない**: 不明点があれば、実装を読む前にユーザーに確認
- **コンテキストを保持**: 読み込んだ内容は次の作業で活用する
- **要約を提供**: 理解した内容を簡潔に説明し、認識の齟齬がないか確認

## 出力形式

実装を理解した後、以下の形式で要約を提供してください：

```
## 実装の理解完了

### 読み込んだファイル
- [ファイルリスト]

### アーキテクチャ/パターン
- [主要な設計パターンや構造]

### 主要な機能/コンポーネント
- [重要な機能やコンポーネントの説明]

### 次のステップ
実装の理解が完了しました。次にどのような作業を行いますか？
```

それでは開始します。参考にしたい実装のファイルパスやパターンを教えてください。
